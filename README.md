## Path Aliases

Section components reference shared utilities via `@/lib/...` and `@/components/ui/...`. When you copy the package into your project:

- Place the contents of `_shared/lib/` at `src/lib/` (or wherever your `@/lib` alias points).
- Place the contents of `_shared/components/ui/` at `src/components/ui/`.
- Place section types from `sections/[id]/types.ts` so the relative `../types` import inside each component resolves.

Alternatively, configure a `_shared` path alias in your bundler — but mounting them at `src/lib` and `src/components/ui` is the simplest approach.

## Test-Driven Development

Each section includes a `tests.md` file with test-writing instructions. For best results:

1. Read `sections/[section-id]/tests.md` before implementing.
2. Write failing tests based on the instructions.
3. Implement the feature to make tests pass.
4. Refactor while keeping tests green.

The test instructions are **framework-agnostic** — they describe WHAT to test, not HOW. Adapt to your testing setup (Jest, Vitest, Playwright, Cypress, RSpec, Minitest, PHPUnit, etc.).
